# ACTION Test Scenarios

## Сценарий 1: Успешный ACTION после LOGIN
**Тип:** Позитивный
**Приоритет:** Critical

**Предусловия:**
- Приложение запущено
- WireMock настроен: POST /auth → 200, POST /doAction → 200
- Токен прошел LOGIN

**Шаги:**
1. Выполнить LOGIN с токеном
2. Отправить POST /endpoint с этим токеном и action=ACTION
3. Проверить что WireMock получил запрос на /doAction

**Ожидаемый результат:**
- HTTP статус 200
- result: "OK"

**Постусловия:**
- LOGOUT

---

2. Выполнить LOGIN с токеном2

- LOGOUT токена1 не влияет на токен2

**Постусловия:**
