# LOGIN Action Test Scenarios

## Сценарий 1: Успешный LOGIN
**Тип:** Позитивный
**Приоритет:** Critical

**Предусловия:**
- Приложение запущено
- WireMock настроен: POST /auth → 200 OK

**Шаги:**
1. Сгенерировать валидный токен
2. Отправить POST /endpoint с token и action=LOGIN
3. Проверить что WireMock получил запрос на /auth с этим токеном

**Ожидаемый результат:**
- HTTP статус 200
- result: "OK"
- Токен сохранен во внутреннем хранилище

**Постусловия:**
- Очистить внутреннее хранилище (LOGOUT)
