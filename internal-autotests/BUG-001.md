# BUG-001: Токен принимает только A-F вместо A-Z

## Статус
🔴 **ОТКРЫТ** | Severity: **BLOCKER**

## Описание
Согласно ТЗ, токен должен состоять из 32 символов формата **A-Z0-9** (26 букв + 10 цифр).  
По факту приложение принимает только **A-F0-9** (hex формат: 6 букв + 10 цифр).

## Воспроизведение

### Ожидаемое поведение (по ТЗ):
```bash
curl -X POST http://localhost:8080/endpoint \
  -H "X-Api-Key: qazWSXedc" \
  -d "token=GHIJKLMNOPQRSTUVWXYZ12345678901&action=LOGIN"

# Ожидается: {"result":"OK"}
```

### Фактическое поведение:
```bash
# Возвращает: {"result":"ERROR","message":"Invalid token format"}
```

### Работает только с A-F:
```bash
curl -X POST http://localhost:8080/endpoint \
  -H "X-Api-Key: qazWSXedc" \
  -d "token=ABCDEF1234567890ABCDEF1234567890&action=LOGIN"

# Возвращает: {"result":"OK"} ✅
```

## Тестовое покрытие

### Тесты с @Disabled (упадут когда баг исправят):
- `shouldAcceptTokenWithFullAlphabet()` - токен с G-Z символами
- `shouldAcceptTokenWithLetterG()` - минимальный тест с буквой G
- `shouldAcceptTokenWithLetterZ()` - граничный тест с буквой Z

### Тесты текущего поведения (проходят):
- `shouldAcceptOnlyHexTokens()` - документирует что работает A-F
- `shouldRejectTokensWithGtoZ()` - документирует что G-Z отклоняется

## Расположение тестов
```
tests/bugs/BugValidationTest.java
```

## Запуск тестов бага
```bash
# Запустить только тесты текущего поведения (без @Disabled)
mvn test -Dtest=BugValidationTest

# Запустить все тесты включая @Disabled (упадут)

```

## Влияние
- **Критичность:** BLOCKER
- **Пользователи:** Не могут использовать токены с символами G-Z
- **Обход:** Использовать только hex токены (A-F0-9)

## Конфигурация

### Текущая (работает с багом):
```java
// TestConfig.java
public static final String VALID_TOKEN_CHARS = "0123456789ABCDEF";
public static final String VALID_TOKEN_PATTERN = "[0-9A-F]{32}";
```

### Правильная (когда баг исправят):
```java
// TestConfig.java
public static final String VALID_TOKEN_CHARS = "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ";
public static final String VALID_TOKEN_PATTERN = "[0-9A-Z]{32}";
```

## Действия после исправления

1. Обновить `TestConfig.java`:
   - `VALID_TOKEN_CHARS` → добавить G-Z
   - `VALID_TOKEN_PATTERN` → изменить на `[0-9A-Z]{32}`

2. Убрать `@Disabled` с тестов в `BugValidationTest.java`

3. Запустить все тесты:
   ```bash
   mvn clean test
   ```

4. Проверить что все 3 теста бага проходят ✅
