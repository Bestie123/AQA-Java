## Подтверждение бага

### Текущее поведение:
```java
// Приложение валидирует токен по паттерну:
Pattern: ^[0-9A-F]{32}$
// Принимает: 0-9, A-F (16 символов)
```

### Ожидаемое поведение (по ТЗ):
```java
// Должно быть:
Pattern: ^[0-9A-Z]{32}$
// Должно принимать: 0-9, A-Z (36 символов)
```

### Разница:
- **Текущее:** 16 символов (0-9, A-F) - hex формат
- **Ожидаемое:** 36 символов (0-9, A-Z) - полный алфавит
