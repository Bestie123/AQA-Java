# Автотесты для Internal Application

Этот проект содержит автоматические тесты для проверки работы веб-приложения Internal Application.

## 📋 Что протестировано

Приложение имеет один эндпоинт с тремя действиями:
- **LOGIN** - вход пользователя в систему
- **ACTION** - выполнение действия (доступно только после входа)
- **LOGOUT** - выход из системы

### Покрытие тестами: **46 автотестов**

✅ **Валидация** (18 тестов) - проверка правильности данных  
✅ **LOGIN** (10 тестов) - проверка входа в систему  
✅ **ACTION** (4 теста) - проверка выполнения действий  
✅ **LOGOUT** (4 теста) - проверка выхода из системы  
✅ **Интеграция** (5 тестов) - проверка полных сценариев работы  
✅ **Баги** (5 тестов) - проверка известных проблем  

## 🚀 Как запустить тесты

### Требования
- Установленная Java 17 или выше
- Установленный Maven 3.6+

### Шаг 1: Запустить приложение
Откройте командную строку в папке проекта и выполните:
```
start-app.bat
```
Приложение запустится на http://localhost:8080

### Шаг 2: Запустить тесты
В той же папке выполните:
```
cd internal-autotests
run-tests.bat
```

### Шаг 3: Посмотреть отчет
После завершения тестов выполните:
```
mvn allure:serve
```
Откроется браузер с красивым интерактивным отчетом.

### Генерация тестов через консоль
Вы можете запускать тесты напрямую через Maven, используя следующие команды:

- **Запуск всех тестов:**
  ```bash
  mvn test
  ```

- **Запуск тестов определённого класса:**
  ```bash
  mvn test -Dtest=LoginTest
  ```

- **Запуск тестов с определённой категорией (например, валидация):**
  ```bash
  mvn test -Dgroups=validation
  ```

- **Генерация отчёта Allure без запуска сервера:**
  ```bash
  mvn allure:report
  ```
  Отчёт будет сгенерирован в `target/site/allure-maven-plugin/index.html`

- **Запуск тестов с параллельным выполнением (если настроено):**
  ```bash
  mvn test -Dparallel=methods -DthreadCount=4
  ```

Для более детальной настройки обратитесь к `pom.xml` проекта.

## 📊 Что показывает отчет

Отчет Allure содержит:
- **Общую статистику** - сколько тестов прошло/упало
- **Графики** - визуальное представление результатов
- **Детали каждого теста** - что проверялось, какой результат
- **Группировку** - тесты сгруппированы по функциям
- **Историю** - можно сравнить с предыдущими запусками

### Как читать отчет

**Зеленый цвет** ✅ - тест прошел успешно  
**Красный цвет** ❌ - тест обнаружил проблему  
**Желтый цвет** ⚠️ - тест пропущен (известная проблема)
